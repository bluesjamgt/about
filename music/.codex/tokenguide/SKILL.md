目標：
1. 只讀這次任務必要的檔案與必要區段，不重掃整包 repo。
2. 優先用精準片段讀取，不用整檔全文；只在需要修改前後文時才擴大區間。
3. 不掃描其他資料夾。
4. 完成後只輸出：修改檔案、修改摘要、驗證結果。
5. 不要執行完整建置；只做瀏覽器可載入與語法檢查。

工作規則：
1. 先用 `Select-String` 或 `rg` 找函式名、state 欄位、DOM selector，再只讀命中的行段。
2. 同一檔案第二次讀取時，優先續讀相鄰區段，不重新從檔頭掃描。
3. 已確認不相關的檔案不要再打開第二次。
4. 驗證先做最小閉環：
   - JS 用 `node --check`
   - CSS 只做括號平衡或必要 selector 驗證
   - 瀏覽器測試只驗核心互動，不重跑整頁流程
5. 同一需求最多保留 1 到 2 個代表性瀏覽器案例：
   - 一個驗主流程
   - 一個驗邊界條件
6. 如果前一輪已測過且這輪沒有動到該區塊，就不要重測。
7. 回覆前只整理：
   - 修改檔案
   - 修改摘要
   - 驗證結果

建議做法：
1. 先建立「關鍵函式名單」再工作，例如 `baseShouldShow`、`renderNoteSwitchboard`、`markerLabels`。
2. 需要跨回合延續時，把這輪已確認的根因、檔案、驗證方式寫進摘要，避免下一輪再讀一次舊上下文。
3. 能用 state 與 DOM 計數驗證的，就不要額外截圖或跑完整輸出流程。
